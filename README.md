1. 프로젝트 개요

1-1. 목표
고양이 이미지와 단순한 카피를 활용한 싱글 페이지 랜딩 페이지를 제작한다.
헤더 내비게이션, 히어로 섹션, 특징 소개, 구독(Subscribe) 섹션, 푸터까지 웹 서비스의 기본 구조를 HTML/CSS로 구현하는 것이 목표였다.
반응형 레이아웃과 팝업(dialog) 구조를 통해, 실제 서비스에 바로 적용 가능한 UI 설계 경험을 쌓는 것을 목표로 했다.

1-2. 기능 (UI 중심)
헤더
좌측 로고(Hodu)와 우측 메뉴(Home / About / Support / Download 버튼) 배치.
모바일 환경에서 사용할 햄버거 메뉴 아이콘(.menu) 추가.

히어로 섹션 (Section)
서비스의 핵심 메시지를 담은 제목과 설명 문단.
“Download” 버튼으로 사용자의 주요 행동(CTA)을 유도.
옆에 box-cat.png 이미지를 배치해, 시각적으로 시선을 끄는 메인 히어로 영역 구성.

콘텐츠 소개 영역 (Figure, Article)
figure : 메인 고양이 이미지(img_4.jpg)와 짧은 텍스트로 서비스/브랜드 분위기를 소개.
article : 제목 + 설명 문단 + 3장의 고양이 이미지(갤러리 형식)를 배치해 이미지 카드형 UI를 구현.

Subscribe 섹션
간단한 설명 문단과 learn more 버튼으로, 추가 정보 페이지 또는 상세 설명으로 이동하게 하는 서브 CTA 영역.

푸터 상단 박스 (.footer-box)
“Subscribe to our Blog post” 제목과 설명,
이메일 입력 필드 + Subscribe 버튼으로 구성된 뉴스레터 구독 UI 구현.

팝업(Dialog) 구조
<dialog> 태그를 사용하여 향후 사용할 모달 팝업의 HTML 뼈대를 구성.
제목, 내용, 버튼으로 이루어진 기본 모달 레이아웃을 만들어 두어, 추후 JavaScript로 열고 닫을 수 있게 할 예정
	
푸터 
좌측 로고와 우측 SNS 아이콘(블로그, 인스타그램, 페이스북, 유튜브)을 정리한 링크 영역.
하단 메뉴(About, Blog, Support, Terms of Use)를 통해 사이트 푸터 내비게이션 구조를 구현.

3. UI, UX 및 디자인 
3-1. 메인 웹페이지 
레이아웃
상단 헤더 – 중앙 메인 콘텐츠 – 하단 푸터로 이어지는 단일 스크롤 레이아웃.
.wrap 컨테이너로 섹션들을 감싸서, 폭을 제한하고 정돈된 느낌을 줌.

히어로 섹션 UX
핵심 카피와 설명 문단, Download 버튼을 한 덩어리(.hero-textbox)로 묶어 사용자가 페이지에 진입하자마자 서비스 소개와 행동 버튼을 동시에 인지할 수 있도록 구성.
오른쪽 고양이 이미지(box-cat.png)로 시선을 끌고, 전체 페이지 톤을 아기자기하고 따뜻한 분위기로 설정.

콘텐츠 구역 (Figure, Article)
figure에 주요 이미지와 짧은 설명을 배치해, “이미지 + 설명 텍스트” 패턴을 연습.
article에는 제목, 강조 텍스트(<strong>dummy text</strong>), 세 장의 이미지(img_1.jpg, img_2.jpg, img_3.jpg)를 나란히 배치해카드형 갤러리 UI를 구성.

구독/설명 섹션
.subscribe에서 긴 설명 문단 후 learn more 버튼을 제공해, 사용자가 흥미를 느끼면 더 자세한 정보를 탐색할 수 있도록 UX 흐름을 만들었다.
푸터
.footer-box에서 이메일 구독 영역과 설명을 먼저 배치하고,
최하단 footer에서 로고와 SNS 아이콘, 텍스트 링크 메뉴를 통해 브랜드 마무리 + 추가 탐색 경로를 제공한다.

3-2. 팝업창
요소 활용
HTML5의 <dialog> 태그를 사용해 팝업 구조를 미리 정의해 두었다.
내부에 <h3>, <p>, <button> 요소로 구성된 <div>를 넣어
알림 제목
설명 텍스트
확인/닫기 버튼 이 기본적으로 갖춰진 상태다.
UX 의도
향후 사용자가 구독을 완료했을 때, 혹은 Download 클릭 후 추가 안내를 보여줄 때, 이 모달을 활용해 명확한 피드백과 다음 행동을 안내할 수 있도록 계획했다.
브라우저 기본 모달이 아닌, CSS로 커스텀 스타일링을 적용해 전체 디자인 톤(Hodu, 고양이, 따뜻한 색감)과 어울리게 만드는 것을 목표로 한다.

4. 핵심 코드 및 배운 점
4-1. CSS 변수 활용
(style.css 내용 기준, 개념적으로 정리)
색상, 폰트 크기, 여백 값 등을 :root에 CSS 변수로 정의해두고 재사용하는 방식으로 스타일을 구성했다.
예: :root {
  --color-primary: #FF7A00;
  --color-text: #333333;
  --font-main: "Noto Sans KR", sans-serif; }
버튼, 텍스트, 배경 등에 같은 변수를 사용함으로써:
디자인 톤을 일관되게 유지할 수 있었고,
나중에 색상이나 폰트를 바꾸더라도 변수만 수정하면 전체 스타일이 함께 바뀌는 관리의 편리함을 체감했다.
이 과정을 통해 **“CSS도 일종의 디자인 토큰 시스템처럼 관리할 수 있다”**는 개념을 이해하게 되었다.

4-2. 반응형 미디어 쿼리
(실제 style.css에서 구현했다는 가정하에 작성)
@media (max-width: …px) 미디어 쿼리를 사용해,
데스크톱과 모바일에서 레이아웃이 자연스럽게 변하도록 설정했다.
주요 반응형 처리 내용:
헤더:
넓은 화면: 로고 + 텍스트 메뉴/버튼이 가로로 배치.
좁은 화면: 텍스트 메뉴를 숨기고, 오른쪽 상단의 햄버거 아이콘(.menu)만 보이게 변경.
히어로 섹션:
데스크톱: 텍스트와 고양이 이미지가 가로로 나란히 배치.
모바일: 세로로 쌓이게 정렬하여 작은 화면에서도 읽기 쉬운 구조로 변경.
article 이미지 3장:
데스크톱: 가로 3열 배치.
모바일: 1~2열로 줄어들어 이미지가 너무 작아지지 않도록 조정.
이 작업을 통해 단순히 줄어드는 페이지가 아니라,
화면 크기에 맞춰 유연하게 재배치되는 레이아웃 설계 방법을 배웠다.

4-3. 폰트 및 웹 접근성
폰트:
기본 웹폰트(또는 Noto Sans 계열)를 사용하여 한국어·영어 모두 가독성을 확보하려고 했다.

웹 접근성 요소:
메인 히어로 제목에 class="sc-a11y"를 적용해, 시각적으로는 숨기되 스크린 리더가 읽을 수 있도록 처리하는 시각장애인 친화적 구조를 연습했다.
모든 이미지에 alt 속성을 넣어, 이미지 의미를 화면 읽기 프로그램이 전달할 수 있도록 했다.
SNS 아이콘 링크에는 aria-label을 사용해 “어떤 서비스로 이동하는 링크인지”를 명확히 표현했다.
<a class="sns-link" href="#" aria-label="Instagram">
<img class="sns-icon" src="instagram.svg" alt="">
</a>

이 과정을 통해 시맨틱 태그 + 대체 텍스트 + 숨김 텍스트가
단순한 형식이 아니라 실제 사용자를 위한 장치라는 점을 이해하게 되었다.

6. 개발하며 느낀 점 
단순한 형태의 랜딩 페이지였지만, 헤더–히어로–콘텐츠–구독–푸터 흐름을 처음부터 끝까지 직접 쌓으면서
“웹페이지 한 장이 어떻게 구조화되는지”를 몸으로 익힐 수 있었다.
처음에는 HTML 태그를 그냥 나열하는 느낌이었지만, 시맨틱 태그 선택, figure·article의 역할, dialog 구조까지 생각하다 보니
“마크업 자체가 정보 구조와 경험을 설계하는 일”**이라는 걸 깨달았다.
CSS 변수, 미디어쿼리, 웹 접근성 요소를 적용해 보면서“보기에 예쁜 페이지”와 “사용자 입장에서 편한 페이지”가
같지 않을 수도 있다는 사실을 느꼈고, 앞으로는 두 가지를 동시에 만족시키는 UI를 설계하는 것을 목표로 삼게 되었다.
무엇보다, 정적인 디자인 시안이 아니라 코드로 살아 움직이는 웹페이지를 만들면서 “디자이너이자 퍼블리셔로서 내가 할 수 있는 일의 범위가 넓어지고 있다”는
성취감을 느낀 프로젝트였다.
